h1. Manifest

The @Manifest.xml@ file (older Config.xml) is the main reference of the ADX.

It contains all metadata, public/private properties and generation rules.
It must be available at the root of the ADX file structure.

The schema of @manifest.xml@ is defined by the "ADCManifest.xsd":http://www.askia.com/Downloads/dev/schemas/adc2.1/ADCManifest.xsd file for ADC and "ADPManifest.xsd":http://www.askia.com/Downloads/dev/schemas/adp1.0/ADPManifest.xsd file for ADP.

h2. Notes about the type format used in the XML

h3. (1) Legal version number

Legal version number using maximum 3 digits and optional alpha, beta or rc version.

*Pattern*
<pre><code>^(\d+)(?:\.(\d+))?(?:\.(\d+))?(?:\.(\d+))?(?:\.(alpha|beta|rc)(\d*))?$</code></pre>

*Examples*
<pre><code>
1
1.0
1.0.1
1.0.1.alpha
1.alpha1
2.12.132.beta25
0.2.rc3
</code></pre>

h3. (2) Number pattern

Based on "xsd:token":http://www.schemacentral.com/sc/xsd/t-xsd_token.html and restricted to a number pattern, it allows the usage of star symbol (*) to indicates an unbounded number.

*Pattern*
<pre><code>^-?(\*|\d+(\.\d+)?)$</code></pre>

*Examples*
<pre><code>
0
0.1
123.456
456
* (unbounded number)
</code></pre>
